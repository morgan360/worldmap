There are many tools that help write, organize and display worlds

But there are limited ways to transfer world data across these domains

OnlyWorlds aims to provide a common standard in world building

  • By proposing a 'blueprint' for creating worlds and their content
  • That supports the broadest range of worlds
  • And is transferable across tools, games, devices and web

The framework includes

  • A substantial database schema
  • Infrastructure for open source development and deployment
  • A website for hosting and managing world files
  • A core set of tools
  • A promise of games

OnlyWorlds aims to always provide

  • An open source, community-driven and future proof data system
  • Freedom to build applications for and worlds with it
  • Ownership and control of worlds to the users

18 April 2024


Database

  • The core framework consists of 18 element Categories, each with a set of attribute groups and fields, many of which link to other elements
  • Content level: basics in place, in need of user feedback
  • Technical level: reworking the legacy SQLite systems into a central open source
  • Aim to coalesce these two into a 1.0 release late 2024

Mobile

  • The mobile app is the first release on the framework
  • It is a prototype that introduces the framework in hope for feedback
  • The app can send its worlds to the website, from where it can be viewed and exported

Infrastructure

  • Functional: user registration, authentication, world api
  • Next work is on profile management, world interfacing
  • Two Unity tools (editor, WebGL) are operational with the api, shortly available
  • Discord server available for feedback and community
