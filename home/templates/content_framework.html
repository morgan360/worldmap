There are many tools that help write, organize and display worlds

But there are limited ways to transfer world data across applications

OnlyWorlds provides a common standard in world building through

  • Proposing a 'blueprint' for worlds and their content
  • Supporting maximum diversity in worlds
  • Allowing transfer across tools, games, devices and websites

The framework includes

  • A universal database schema
  • Infrastructure for open source development and deployment
  • A website for hosting and managing worlds
  • A set of core world creation tools

This database schema breaks down world data through several levels:

  • 18 essential element categories
  • Groups and fields to define and link them
  • Types and subtypes to classify and organize them
  • An open source, community-driven and future proof data system
  • Freedom to build applications and tools that expand the framework
  • Ownership and control of worlds remains yours

I have been developing world building tools in various capacities since 2011. Much of this in Unity, with a highlight in WorldSmith in 2018: An open source world engine where I recreated ancient Sicily of ~500BC with an economy simulation and game built on top.
My interests include ((tools for (simulation of)) worlds for) games and history: political, economy, military.
One of my favorite creative puzzles is breaking down real world things into usable data. In my master's thesis I focused on the translation of political dynamics into world building structures.
